Multi User Test Runner

Multi User Test Runner is a custom JUnit (4.12 or newer) test runner for testing authorization with multiple different users. The library requires only minimal configuration to be able run tests with many different users and roles. It is easy to verify that your system denies and permits the access for correct users even in more complex scenarios where the authorization depends on multiple users and their roles.

Usage

Configure the base test class:

  1. Create a configured abstract class by extending {@link fi.vincit.multiusertest.test.AbstractUserRoleIT} class and implement methods. This will be your base class for your test class.
  2. Configure runner and default expected annotation with {@link fi.vincit.multiusertest.annotation.MultiUserTestConfig}
  3. Create a test class which is extended from your configured class.
  4. Add {@link fi.vincit.multiusertest.annotation.TestUsers} annotation for your test class
Write the tests:
  1. Write test methods
  2. Add e.g. authentication().expect(toFail(ifAnyOf("user:user"))); before method to test to mark which roles/users are expected to fail
